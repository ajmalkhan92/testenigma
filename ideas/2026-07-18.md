# Idea digest — 2026-07-18

10 topics for grooming. Each has a code angle and a visual/interactive element
that would work well embedded in the post itself.

1. **Catching LLM Hallucinations with a Simple Regression Harness**
   Run the same prompt N times across model versions, diff the outputs,
   flag drift automatically.
   - Code: Python + pytest harness that snapshots and diffs LLM outputs.
   - Visual: pipeline diagram (prompt → N runs → diff → flag) + a results
     table with diffs highlighted inline.

2. **Self-Healing Selenium Tests: Letting AI Fix Broken Locators**
   When a CSS/XPath selector breaks from a UI change, have an LLM propose
   a new locator from the current DOM.
   - Code: a Selenium wait-wrapper that calls an LLM on locator failure.
   - Visual: before/after DOM diagram showing the broken vs. healed locator.

3. **Prompt Regression Testing: Treating Prompts Like Code**
   Version prompts, snapshot their outputs, diff on every prompt change —
   same discipline as diffing code.
   - Code: pytest snapshot-testing setup for prompts.
   - Visual: git-diff-style colored side-by-side output comparison.

4. **Building an LLM-as-Judge Eval Pipeline (and Why You Shouldn't Trust It Blindly)**
   Use one LLM to grade another's outputs against a rubric — and show
   where judge bias and inconsistency creep in.
   - Code: eval script scoring outputs, computing agreement vs. a human
     baseline.
   - Visual: scatter plot of LLM-judge score vs. human score, disagreement
     zone highlighted.

5. **AI Test Case Generation from User Stories: A Practical Walkthrough**
   Feed a user story to an LLM, generate Gherkin/BDD scenarios, then
   critique the output quality honestly.
   - Code: prompt template + parser turning a story into Gherkin.
   - Visual: tabbed/toggle comparison of "AI-generated" vs.
     "human-reviewed" scenarios.

6. **Chaos Engineering for AI Agents: What Happens When Tools Fail Mid-Task**
   Inject random tool/API failures into a multi-step agent run and observe
   whether it recovers.
   - Code: fault-injection decorator wrapping an agent's tool calls.
   - Visual: agent task-graph/state diagram with failure injection points
     marked.

7. **Flaky Test Detection Using Statistics — and Where AI Actually Helps**
   Classic flakiness scoring from rerun data, then use embeddings to
   cluster failure messages by likely root cause.
   - Code: flakiness-score script + embedding-based clustering of error
     logs.
   - Visual: flakiness heatmap per test + 2D cluster scatter plot of
     failure messages.

8. **Evaluating RAG Pipelines: Precision, Recall, and the Metrics Nobody Talks About**
   Test retrieval quality and generation quality separately instead of
   judging a RAG system as one opaque blob.
   - Code: retrieval precision/recall eval against a labeled set +
     faithfulness scoring.
   - Visual: bar chart comparing metrics across chunk-size/top-k configs.

9. **Mutation Testing Meets AI: Can an LLM Catch When You've Broken Your Own Code?**
   Deliberately introduce bugs (mutation testing) and benchmark whether an
   AI code reviewer catches them.
   - Code: mutation runner + AI code-review call per mutant, detection
     rate tallied.
   - Visual: scorecard/gauge comparing AI detection rate vs. human
     baseline.

10. **Snapshot Testing AI-Generated UI Components**
    When AI generates a React/UI component, how do you regression-test
    what it actually renders?
    - Code: Playwright visual-regression setup across prompt iterations.
    - Visual: before/after slider with pixel-diff highlighting.
